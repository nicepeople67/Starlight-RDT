Setup guide

Get connected
in minutes.

Step-by-step instructions for every platform. Takes about 2 minutes from download to connected.

1

Download and run the installer

Download deltardt-setup.exe and double-click it. Windows SmartScreen may warn about an unknown publisher; click "More info" > "Run anyway". This is normal for unsigned software.

Can't install? Use the portable .zip instead; extract and double-click deltardt.exe, no installation needed, no admin rights.

2

Allow through Windows Firewall

A firewall prompt appears on first run. Click Allow access for both Private and Public networks. This lets the agent reach the relay server outbound; no inbound ports are opened.

If the prompt doesn't appear, go to Windows Security > Firewall > Allow an app and add DeltaRDT manually.

3

Your session code appears

A tray icon appears in the system tray (bottom-right). The DeltaRDT window shows your session code; e.g. AB3-X92. Share this with whoever needs to connect.

Codes refresh every 24 hours. Right-click the tray icon > New Code to generate a fresh one anytime.

4

Optional: start with Windows

Right-click the tray icon > Start with Windows to always have DeltaRDT running in the background so you can connect remotely even after a reboot.

5

Share your code and connect

On the Chromebook (or any browser), click Connect Now in the nav bar. Enter the session code and click Connect. You'll see the remote desktop in ~2 seconds.

> Open Connect page
1

Download and open the .dmg

Download DeltaRDT.dmg open it, and drag DeltaRDT into your applications folder. Then launch it from Applications or Spotlight.

macOS may say the app is from an "unidentified developer". Go to System Settings > Privacy & Security and click "Open Anyway".

2

Grant Screen Recording permission

On first launch macOS prompts for Screen Recording access. If you miss the prompt, go to System Settings > Privacy & Security > Screen Recording and enable DeltaRDT.

3

Grant Accessibility permission

To allow keyboard and mouse input from the remote side, also enable DeltaRDT under Privacy & Security > Accessibility. Both permissions are needed for full control.

4

Session code in the menu bar

DeltaRDT appears in the menu bar (top-right). Click it to see your code. Share it and the viewer connects from Connect Now.

> Open Connect page
1

Install Python dependencies

sudo apt update && sudo apt install python3-pip
pip3 install mss pyautogui websockets Pillow

On Fedora/RHEL: sudo dnf install python3-pip · On Arch: sudo pacman -S python-pip

2

Download and run the agent

Option A; .deb package (Ubuntu/Debian):

sudo dpkg -i deltardt_1.0.0_amd64.deb
deltardt

Option B; Python script (any distro):

python3 vnc_server.py
3

Wayland users; set DISPLAY

DISPLAY=:0 python3 vnc_server.py

Pure Wayland without XWayland may limit screen capture. Ubuntu 22+ has XWayland enabled by default.

4

Session code in terminal or GUI

The agent prints your code to the terminal and opens a small GTK window. Share the code and connect from the browser.

> Open Connect page
1

No download needed on the viewer side

The Connect Now viewer runs entirely in your browser; on a Chromebook, phone, tablet, or another PC. Just open this site and click Connect Now in the nav.

> Open Connect page
2

Enter the session code

Type the session code shown on the host PC. The format is 6 characters like AB3X92. Leave the relay URL as the default unless you're self-hosting.

The relay field defaults to wss://relay.deltardt.app. Only change this if you're running your own relay server.

3

You're in; full keyboard & mouse control

The remote desktop appears in the browser. Click to move the mouse, type to send keystrokes. The toolbar gives you Ctrl+Alt+Del, clipboard sync, fit-to-screen, and fullscreen.

On Chromebook: press Ctrl+Shift+F to go fullscreen. All keyboard shortcuts are forwarded correctly to the remote PC.

4

Adjust quality for your connection speed

Use the Quality dropdown in the viewer toolbar. Low works great on cell service or garbage hotel WiFi. Full is best on a fast (500mbps+) home network.

1

Why self-host?

Run the relay on your own VPS for zero third-party involvement or many simultaneous sessions. A free-forever Oracle server handles hundreds of concurrent sessions.

2

Clone the repo and install

git clone https://github.com/nicepeople67/Delta-RDT
cd deltardt
pip3 install websockets
3

Run the relay server

# Start relay; use nginx in front for TLS
python3 agent/relay_server.py --ws-port 443 --api-port 444

Use nginx as a reverse proxy to handle TLS (Let's Encrypt). This gives you wss:// required for HTTPS pages.

4

Register a session

curl -X POST http://your-server:444/api/sessions \
-H 'Content-Type: application/json' \
-d '{"host":"127.0.0.1","port":5900,"label":"My PC"}'
# > {"token": "abc123def456"}

Enter the token as your session code in the viewer, and set the relay URL to wss://your-server.

5

Point the agent at your relay

python3 agent/vnc_server.py --relay wss://your-server --port 5900

Your screen data now flows entirely through your own server. DeltaRDT's relay never sees it.

Features

Built right,
not bolted on.

End-to-end encrypted

All traffic is TLS-encrypted via the relay. Your screen data is never stored or inspected on our servers.

Hextile encoding

Our VNC server uses Hextile tile compression; only changed screen regions are sent each frame.

<

Works across networks

Connect from a Chromebook at a coffee shop to a PC at home. Different networks, different ISPs; no problem.

Clipboard sync

Copy on the remote, paste in the browser. Or vice versa. Bidirectional text clipboard sync built in.

Full keyboard & mouse

Every key; Ctrl, Alt, Win/Cmd, F-keys, special characters; forwarded exactly to the remote machine.

Open source & self-hostable

MIT licensed. Audit the code, run your own relay, or contribute back. No black boxes.
